# 대화 로그 (세션 기록)
- 저장일시: 2026-03-10
- 프로젝트: ggrigggri_compose

## 1) 요청 배경
- 사용자: FCM 기능이 AI로 작성되어 동작 이해가 어렵고, 의도한 시나리오(오늘의 질문/요청/응답 대상자별 알림)와 현재 코드의 부족점을 점검 요청.

## 2) 코드 점검 결과(핵심)
- 앱 수신부(FirebaseMessagingService, NotificationHandler, DisplayManager)는 존재.
- 서버 발송부(Cloud Functions) 실질 구현이 없어서 대상자별 푸시 발송이 동작하지 않던 상태.
- 알림 클릭 라우팅(인텐트 extra 소비) 미구현 지점 확인.
- 토큰 관리/다중기기 처리 관련 구조 리스크 확인.

## 3) 구현 작업
- 파일 수정: `functions/src/index.ts`
- 추가한 함수(2nd gen Firestore trigger):
  1. `notifyDailyQuestionCreated`
     - 트리거: `question_data/{questionId}` 생성
     - 대상: 그룹 멤버 전체
     - 데이터: `type=daily_question`, `groupId`, `questionId`, `questionListId`
  2. `notifyRequestCreated`
     - 트리거: `request_data/{requestId}` 생성
     - 대상: 요청자 제외 그룹 멤버
     - 데이터: `type=request`, `groupId`, `requestId`, `requesterUserId`
  3. `notifyResponseCreated`
     - 트리거: `request_data/{requestId}/response_data/{responseId}` 생성
     - 대상: 요청자 1명
     - 데이터: `type=response`, `groupId`, `requestId`, `responseId`, `requesterUserId`, `responderUserId`
- 공통 유틸:
  - 사용자 토큰 수집/정규화(`_userFcmToken`)
  - 중복 제거
  - `sendEachForMulticast` 발송
  - data payload 문자열화

## 4) 배포 과정 이슈 및 해결
- 경로 이슈:
  - `firebase.json`의 functions source 경로 불일치로 `package.json` 찾기 실패(ENOENT).
  - 수정: `source = ggrigggri_compose/functions`
- 린트 이슈:
  - `max-len`, `object-curly-spacing` 오류 해결 후 `npm run lint`, `npm run build` 통과.
- 2nd gen 초기 권한 전파 이슈:
  - Eventarc Service Agent 권한 전파 지연으로 초기 배포 실패 로그 발생.
  - 재배포 후 함수 3개 생성 성공.

## 5) 최종 상태
- 배포 성공 확인:
  - `notifyDailyQuestionCreated(us-central1)` ACTIVE
  - `notifyRequestCreated(us-central1)` ACTIVE
  - `notifyResponseCreated(us-central1)` ACTIVE
- 현재 로그에 보이는 초기 Permission denied 항목은 과거 실패 이력이며,
  이후 ACTIVE 생성 로그가 최종 성공 상태를 의미.

## 6) 다음 테스트 가이드
- A/B 같은 그룹에서 시나리오 검증:
  1. A가 요청 생성 -> B(본인 제외 멤버) 수신
  2. B가 응답 생성 -> A(요청자) 수신
  3. 질문 생성 이벤트 -> 그룹 전체 수신
- 함수 로그 확인:
  - `firebase functions:log --only notifyDailyQuestionCreated --lines 50`
  - `firebase functions:log --only notifyRequestCreated --lines 50`
  - `firebase functions:log --only notifyResponseCreated --lines 50`
