## 📱 프로젝트 개요
- **프로젝트명**: GGriggri (끼리끼리)

## 🏗️ 아키텍처 및 설계

### Clean Architecture 적용
프로젝트를 **Domain**, **Data**, **Feature** 모듈로 명확히 분리하여 의존성 역전 원칙(DIP)을 준수했습니다.

- **Domain Layer**: 비즈니스 로직과 인터페이스 정의 (의존성 없음)
  - Repository 인터페이스
  - Domain Model
  - Use Case 로직

- **Data Layer**: 데이터 소스 구현 및 Repository 구현체
  - Firebase Firestore/Storage 연동
  - DataStore를 통한 로컬 데이터 관리
  - DTO ↔ Domain Model 매핑

- **Feature Layer**: UI 및 ViewModel
  - Jetpack Compose 기반 UI
  - MVVM 패턴 적용

### 모듈 구조
```
app/
├── feature/
│   ├── ggriggri_ui/      # 메인 기능 UI
│   └── common_ui/         # 공통 UI 컴포넌트
├── data/                  # 데이터 레이어
├── domain/                # 도메인 레이어
└── core/                  # 공통 유틸리티
```

## 🔧 주요 기술 스택

### 의존성 주입: Hilt 마이그레이션
**Factory 패턴에서 Hilt로 업그레이드**

#### Before (Factory 패턴)
- ViewModel 생성 시 수동으로 Factory 클래스 작성 필요
- 의존성 주입 코드가 반복적이고 보일러플레이트 코드 증가
- 테스트 시 Mock 객체 주입이 복잡함

#### After (Hilt 적용)
- `@HiltViewModel` 어노테이션으로 ViewModel 자동 주입
- `@Module`, `@Provides`, `@Binds`를 통한 체계적인 의존성 관리
- `@InstallIn`으로 컴포넌트 스코프 명확히 정의
- WorkManager와의 통합을 위한 `HiltWorkerFactory` 적용

**주요 구현 사항:**
```kotlin
// Application 레벨 Hilt 설정
@HiltAndroidApp
class ggriggriAplication : Application(), Configuration.Provider {
    @Inject
    lateinit var workerFactory: HiltWorkerFactory
    
    override val workManagerConfiguration: Configuration
        get() = Configuration.Builder()
            .setWorkerFactory(workerFactory)
            .build()
}

// ViewModel 주입
@HiltViewModel
class HomeViewModel @Inject constructor(
    private val sessionManager: SessionManager,
    private val userRepository: UserRepository,
    // ... 기타 의존성
) : ViewModel()

// Repository 바인딩
@Module
@InstallIn(ViewModelComponent::class)
abstract class FirestoreRepositoryModule {
    @Binds
    abstract fun bindUserRepository(
        firestoreUserRepositoryImpl: FirestoreUserRepositoryImpl
    ): UserRepository
}
```

**개선 효과:**
- 코드 가독성 향상 및 보일러플레이트 코드 70% 감소
- 컴파일 타임 의존성 검증으로 런타임 오류 방지
- 테스트 용이성 향상 (Mock 객체 주입 간소화)

### 백그라운드 작업: WorkManager + Hilt
- **HiltWorker**를 통한 의존성 주입 지원

- 네트워크 제약 조건 설정

**구현 예시:**
```kotlin
@HiltWorker
class DailyQuestionWorker @AssistedInject constructor(
    @Assisted appContext: Context,
    @Assisted workerParams: WorkerParameters,
    private val sessionManager: SessionManager,
    private val questionRepository: QuestionRepository,
) : CoroutineWorker(appContext, workerParams)
```

## 📊 기술적 하이라이트

### 1. 반응형 프로그래밍
- **StateFlow/Flow**를 통한 데이터 스트림 관리
- **collectAsStateWithLifecycle**로 생명주기 인식 상태 수집
- **Sealed Class**를 통한 상태 관리 (`DataResourceResult`)

**상태 관리 예시:**
```kotlin
sealed class DataResourceResult<out T> {
    data object Loading: DataResourceResult<Nothing>()
    data class Success<out T>(val data: T): DataResourceResult<T>()
    data class Failure(val exception: Throwable): DataResourceResult<Nothing>()
}
```

### 2. 에러 처리
- `runCatching`을 통한 안전한 예외 처리
- `DataResourceResult`로 통일된 에러 상태 관리
- 사용자 친화적인 에러 메시지 표시

### 3. 성능 최적화
- **이미지 압축**: 업로드 전 자동 압축으로 네트워크 사용량 감소
- **Coil 캐싱**: 메모리/디스크 캐시로 이미지 로딩 최적화
- **LazyColumn**: 대용량 리스트 렌더링 최적화

### 4. 타입 안전성
- Kotlin Serialization 기반 타입 안전 네비게이션
- Sealed Interface를 통한 타입 안전한 라우팅
- 컴파일 타임 타입 체크
